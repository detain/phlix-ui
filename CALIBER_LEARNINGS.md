# Caliber Learnings

Accumulated patterns and anti-patterns from development sessions.
Auto-managed by [caliber](https://github.com/caliber-ai-org/ai-setup) — do not edit manually.

- **[convention]** The consumer `phlix-server/web-ui/package.json` pins `@phlix/ui` by git tag (`"github:detain/phlix-ui#vX.Y.Z"`), NOT by npm version. A release is incomplete until you: bump `package.json` version → update CHANGELOG → rebuild+commit `dist/` → merge → `git tag -a vX.Y.Z <commit> -m ...` → `git push origin vX.Y.Z` → bump the `#vX.Y.Z` ref in the consumer's package.json. Skipping the tag/consumer-bump means the new UI never ships.
- **[pattern]** Release/merge flow in this repo: branch → commit → `git pull --all` → `git push -u origin <branch>` → `gh pr create` → `gh pr merge --squash --delete-branch` → `git checkout master && git pull`.
- **[env]** Prefix git/gh shell blocks with `unset GITHUB_TOKEN` — pushes use the SSH remote (`git@github.com:detain/...`) and a stale `GITHUB_TOKEN` interferes. Commit scripts also probe `command -v caliber` and run `caliber refresh --quiet` before staging.
- **[convention]** DOM-free windowing/paging math lives in `src/components/virtual-grid.ts` as pure exported functions (e.g. `computeWindow`, `effectiveItemCount`, `shouldLoadMore`) and is unit-tested directly in `virtual-grid.test.ts` — jsdom has no layout, so `MediaGrid.vue` only measures and delegates the arithmetic. Add new grid logic as a pure helper there with its own test rather than testing through the component.
- **[gotcha]** `npm run lint` and `npx eslint` do NOT work in this repo — eslint is not installed (`sh: 1: eslint: not found`) and there is no `eslint.config.js` (ESLint v9+ flat-config). Don't waste a step trying to lint changed files; the real gates are `npx vue-tsc --noEmit` (typecheck) + `npx vitest run`.
- **[gotcha]** A green `npx vitest run` does NOT mean the build is clean — vitest does not type-check mocks, but `vue-tsc --noEmit` does. Test mock return values must match the real API method signatures (e.g. the install POST resolves `{ manifest, sources }`, so mock resolutions must include `sources`). Always run the typecheck after editing `.test.ts` files, not just the tests.
- **[gotcha]** When checking a command's exit with `cmd 2>&1 | tail -N; echo "EXIT: $?"`, the `$?` is `tail`'s status (always 0), NOT the command's — so `EXIT: 0` is misleading even when `vue-tsc` printed `error TS...` lines above. Grep the output for `error TS`, or run `npx vue-tsc --noEmit` unpiped, to judge pass/fail.
- **[gotcha]** The phlix-ui CI workflow (`.github/workflows/ui-ci.yml`) is DORMANT — it only triggers on `workflow_dispatch` (the `pull_request`/`push` triggers are commented out, no runner is wired). So `gh pr checks <n>` returns "no checks reported" and `statusCheckRollup` is empty `[]` for every PR. Do NOT poll/wait for CI on phlix-ui PRs — there is no gate; merge once local `vue-tsc --noEmit` + `vitest run` are green. The real gates are local, not GitHub Actions.
- **[pattern]** Component tests in `src/pages/**/*.test.ts` build their API client by injecting a `makeClient`/`mountPage` fake (`{ get, post, put, patch, delete }` of `vi.fn()`) via the `client` prop, plus a `job(overrides)` factory for scan-job rows. To test a new rendered state, override the relevant field in the factory and assert on `data-testid` hooks (`status-${id}`, `progress-${id}`) — follow the existing fixtures rather than mounting against the real `api` singleton.
- **[gotcha]** In `Player.vue`, the `ended` media event does NOT reliably co-fire a `pause` event (Safari notably fires `ended` only) — so the play-state watcher that reveals chrome on stop may never run, leaving end-of-video controls + the bottom-right Up-next card hidden behind the idle auto-hide until a pointer move. Any chrome/visibility logic gated on `player.playing` must be pinned explicitly in `onEnded()` (call `clearIdle()` + `showChrome.value = true`), not left to the play/pause watcher.
- **[pattern]** Test `Player.vue` chrome auto-hide/visibility by driving the raw video element with `video.dispatchEvent(new Event('play'|'pause'|'ended'))` under `vi.useFakeTimers()` + `vi.advanceTimersByTime(...)`, then asserting on `w.classes()` for `is-chrome-hidden` (the idle-hide toggles that class on the root). Pass `idleTimeout` as a mount prop to control the timer window.
- **[pattern]** Typecheck alone with `npx vue-tsc --noEmit` (or `npm run typecheck`); run the full suite with `npx vitest run`. Run an affected subset by passing file paths, e.g. `npx vitest run src/stores/useMediaStore.test.ts`.
- **[convention]** Media-query serialization has ONE source of truth: `buildMediaQuery`/`buildMediaUrl` in `src/api/media-query.ts`. Do NOT add a local query-param builder inside `useMediaStore.ts` (or anywhere) — a duplicated serializer silently drifts and drops params (the `match` and `actors` filters were dropped this way, so the grid stayed at the full unfiltered count). When adding a filter param, add it to `buildMediaQuery` only, then have the store call the shared builder for BOTH the request URL and the `cacheKey`.
- **[gotcha]** `useMediaStore`'s `cacheKey` is derived from the same serializer as the request URL. If a filter param isn't serialized there, two different filter states produce the SAME cache key — toggling the filter re-serves the cached (unfiltered) page instead of refetching. Any new filter must reach the serializer or it both fails to filter AND collides in cache.
- **[pattern]** `useMediaStore.test.ts` asserts request params by stubbing the global fetch (`vi.stubGlobal('fetch', fetchMock)`) and reading the URL off `fetchMock.mock.calls[N][0]` — the store calls raw `fetch`, NOT the injected ApiClient, so to prove a param reaches the server: set it via a store action (`setMatchStatus`/`setActors`), `await s.fetchMedia('')`, then assert `String(fetchMock.mock.calls[0][0])` contains it. Round-tripping through `queryParams`/`toQuery` alone does NOT prove the request carries it.

- **[fix]** When a `vi.fn` async mock returns DIFFERENT object shapes per branch (e.g. `{updated,failed}` vs `{plugin}` vs `{manifest,sources}`), `vue-tsc` infers an over-narrow union whose members are `never[]`, and a later branch like `return { updated: [{name,from,to}], failed: [] }` then fails to assign. Fix by annotating the mock's return type to widen it: `vi.fn(async (endpoint: string): Promise<Record<string, unknown>> => { ... })`. Vitest passes either way; only the typecheck catches this.
