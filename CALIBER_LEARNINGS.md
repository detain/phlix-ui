# Caliber Learnings

Accumulated patterns and anti-patterns from development sessions.
Auto-managed by [caliber](https://github.com/caliber-ai-org/ai-setup) — do not edit manually.

- **[pattern]** Typecheck alone with `npx vue-tsc --noEmit` (or `npm run typecheck`); run the full suite with `npx vitest run` (or `npm run test:run`). Run an affected subset by passing file paths, e.g. `npx vitest run src/utils/sortTitle.test.ts src/api/libraries.test.ts`.
- **[convention]** The consumer `phlix-server/web-ui/package.json` pins `@phlix/ui` by git tag (`"github:detain/phlix-ui#vX.Y.Z"`), NOT by npm version. A release is incomplete until you: bump `package.json` version (`npm version X.Y.Z --no-git-tag-version`) → update CHANGELOG → rebuild+commit `dist/` → merge → `git tag -a vX.Y.Z <commit>` → `git push origin vX.Y.Z` → bump the `#vX.Y.Z` ref in the consumer's package.json. Skipping the tag/consumer-bump means the new UI never ships.
- **[gotcha]** `npm run lint` and `npx eslint` do NOT work in this repo — eslint is installed (v10) but there is no `eslint.config.js` (`ESLint couldn't find an eslint.config.(js|mjs|cjs) file`), so every invocation errors out. Don't waste a step trying to lint changed files; the real gates are `npx vue-tsc --noEmit` (typecheck) + `npx vitest run`.
- **[fix]** When a `vi.fn` async mock returns DIFFERENT object shapes per branch (e.g. `{updated,failed}` vs `{plugin}` vs `{manifest,sources}`), `vue-tsc` infers an over-narrow union whose members are `never[]`, and a later branch like `return { updated: [{name,from,to}], failed: [] }` then fails to assign. Fix by annotating the mock's return type to widen it: `vi.fn(async (endpoint: string): Promise<Record<string, unknown>> => { ... })`. Vitest passes either way; only the typecheck catches this.
- **[gotcha]** When checking a command's exit with `cmd 2>&1 | tail -N; echo "EXIT: $?"`, the `$?` is `tail`'s status (always 0), NOT the command's. Grep the output for `error TS`, or run `npx vue-tsc --noEmit` unpiped, to judge pass/fail.
- **[gotcha]** The phlix-ui CI workflow (`.github/workflows/ui-ci.yml`) is DORMANT — it only triggers on `workflow_dispatch` (the `pull_request`/`push` triggers are commented out; the `test` job runs typecheck+build+test:run but nothing invokes it). So `gh pr checks <n>` returns "no checks reported" and `statusCheckRollup` is empty `[]`. Do NOT poll/wait for CI on phlix-ui PRs — merge once local `vue-tsc --noEmit` + `vitest run` are green. The real gates are local.
- **[pattern]** Pure, side-effect-free utilities (sorting/serialization helpers) go in `src/utils/` (or `src/api/`) and are exported from `src/index.ts` for consumers, then unit-tested directly via `it.each` parity tables. When mirroring server behavior (e.g. `SortTitle` article stripping), keep the article/rule set byte-for-byte in step and assert the shared constant in a test so drift is caught.
- **[convention]** Media-query serialization has ONE source of truth: `buildMediaQuery`/`buildMediaUrl` in `src/api/media-query.ts`. Do NOT add a local query-param builder inside `useMediaStore.ts` (or anywhere) — a duplicated serializer silently drifts and drops params. When adding a filter param, add it to `buildMediaQuery` only, then have the store call the shared builder for BOTH the request URL and the `cacheKey`.
- **[gotcha]** A FilterBar/store test that asserts on `queryParams`/`toQuery` does NOT prove a param reaches the server. To prove it, stub the global fetch (`vi.stubGlobal('fetch', fetchMock)`), set the param via a store action, `await s.fetchMedia('')`, then assert `String(fetchMock.mock.calls[0][0])` contains it — the store calls raw `fetch`, not the injected ApiClient.
- **[convention]** The consumer `phlix-server/web-ui/package.json` pins `@phlix/ui` by git tag (`"github:detain/phlix-ui#vX.Y.Z"`), NOT by npm version. A release is incomplete until you: bump `package.json` version (`npm version X.Y.Z --no-git-tag-version`) → update CHANGELOG → rebuild+commit `dist/` → merge → tag the squash-merge commit → `git push origin vX.Y.Z` → bump the `#vX.Y.Z` ref in the consumer's package.json. Skipping the tag/consumer-bump means the new UI never ships.
- **[convention]** phlix-ui release tags are LIGHTWEIGHT, not annotated. Create them as `git tag vX.Y.Z <squash-merge-commit>` then `git push origin vX.Y.Z` (verify with `git ls-remote --tags origin vX.Y.Z`). `git for-each-ref refs/tags/vX.Y.Z --format='%(objecttype)'` reports `commit` for existing tags — don't "fix" them to `-a` annotated tags to match other repos.
- **[pattern]** Release/merge flow: branch → commit → `git push -u origin <branch>` → `gh pr create` → `gh pr merge --squash --delete-branch` → `git checkout master && git pull --ff-only`. Tag + push the tag afterward.
- **[env]** Prefix git/gh shell blocks with `unset GITHUB_TOKEN` — pushes use the SSH remote (`git@github.com:detain/...`) and a stale `GITHUB_TOKEN` interferes. A caliber pre-commit hook auto-runs on every commit (`caliber: refreshing docs...`) and leaves no extra uncommitted changes — expect it, don't treat its output as an error.
- **[gotcha]** `npm run lint` / `npx eslint` do NOT work in this repo — there is no `eslint.config.js` (flat-config), so every invocation errors out regardless of whether eslint is installed. The real gates are `npx vue-tsc --noEmit` (typecheck, or `npm run typecheck`) + `npx vitest run` (or `npm run test:run`). Run an affected subset by passing file paths.
- **[gotcha]** A green `npx vitest run` does NOT mean the build is clean — vitest does not type-check mocks, but `vue-tsc --noEmit` does. Test mock return values/Response shapes must match the real signatures. Always run the typecheck after editing `.test.ts` files, not just the tests.
- **[gotcha]** The phlix-ui CI workflow (`.github/workflows/ui-ci.yml`) is DORMANT — it only triggers on `workflow_dispatch` (the `pull_request`/`push` triggers are commented out, no runner wired). So `gh pr checks <n>` returns "no checks reported" and `statusCheckRollup` is empty `[]`. Do NOT poll/wait for CI on phlix-ui PRs — merge once local `vue-tsc --noEmit` + `vitest run` are green.
- **[gotcha]** When converting a helper from a raw `fetch` call to going through `ApiClient`, its existing `fetch` mocks BREAK silently (assert mismatch, not a clear error). ApiClient reads `response.status`, `response.headers.get('content-type')`, and `response.json()/.text()` — not just `{ ok, json }`. Rebuild the mock as a Response-shaped object: `{ ok, status, headers: { get: () => 'application/json' }, json, text }`.
- **[convention]** `new ApiClient()` with no explicit `tokenStore` defaults to `LocalStorageTokenStore` in the browser (no-op only under SSR / no `window`), so it auto-sends the logged-in user's Bearer token. Any new client (the `api` singleton, store-created clients, page helpers) is authenticated by default — pass an explicit no-op `tokenStore` ONLY when you deliberately want an unauthenticated request.
- **[convention]** The hub's server-claim endpoint `POST /api/v1/server-claims/claim` requires an `Accept-Phlix-Protocol: v1` header IN ADDITION to the Bearer token. The generic `ApiClient` does not send that protocol header, so claim-style hub calls use a dedicated raw-`fetch` helper (e.g. `claimServer`) that adds both headers itself.
- **[convention]** Media-query serialization has ONE source of truth: `buildMediaQuery`/`buildMediaUrl` in `src/api/media-query.ts`. Do NOT add a local query-param builder anywhere — a duplicated serializer silently drifts and drops params (`match`/`actors` were dropped this way). Add new filter params to `buildMediaQuery` only, then have the store use the shared builder for BOTH the request URL and the `cacheKey` (an unserialized param both fails to filter AND collides in cache).
- **[gotcha]** A FilterBar/store test asserting on `queryParams`/`toQuery` does NOT prove a param reaches the server. The store calls raw `fetch`, not the injected ApiClient — to prove it, `vi.stubGlobal('fetch', fetchMock)`, set the param via a store action, `await s.fetchMedia('')`, then assert `String(fetchMock.mock.calls[0][0])` contains it. To assert the auth header, seed `localStorage.setItem(ACCESS_TOKEN_KEY, ...)` and read `fetchMock.mock.calls[0][1].headers`.
- **[pattern]** Component tests in `src/pages/**/*.test.ts` inject a `makeClient`/`mountPage` fake (`{ get, post, ... }` of `vi.fn()`) via the `client` prop, plus factory functions for fixture rows. Override a field in the factory and assert on `data-testid` hooks. To disambiguate two buttons sharing a label (e.g. a header and a modal both labeled "Add server"), submit via dispatching a `submit` Event on the `<form>` element rather than clicking the button.
- **[pattern]** Pure, side-effect-free utilities (sorting/serialization helpers) go in `src/utils/` (or `src/api/`), are exported from `src/index.ts` for consumers, and are unit-tested directly via `it.each` parity tables. When mirroring server behavior (e.g. `SortTitle` article stripping), keep the rule set byte-for-byte in step and assert the shared constant in a test so drift is caught.
- **[pattern]** DOM-free windowing/paging math lives in `src/components/virtual-grid.ts` as pure exported functions tested directly — jsdom has no layout, so `MediaGrid.vue` only measures and delegates. Add new grid logic as a pure helper there with its own test.
- **[fix]** When a `vi.fn` async mock returns DIFFERENT object shapes per branch, `vue-tsc` infers an over-narrow union (members `never[]`) and later branches fail to assign. Fix by widening the mock's return type: `vi.fn(async (endpoint: string): Promise<Record<string, unknown>> => { ... })`. Vitest passes either way; only the typecheck catches this.
- **[gotcha]** In `Player.vue`, the `ended` media event does NOT reliably co-fire a `pause` event (Safari fires `ended` only) — chrome/visibility logic gated on `player.playing` must be pinned explicitly in `onEnded()` (call `clearIdle()` + `showChrome.value = true`), not left to the play/pause watcher. Test by dispatching raw `new Event('play'|'pause'|'ended')` under `vi.useFakeTimers()` and asserting `is-chrome-hidden` on `w.classes()`.
- **[env]** PHP one-shot scripts run over SSH on `hub.phlix` that open a DB connection fail with `SQLSTATE[HY000] [1045] Access denied for user 'phlix'@'localhost' (using password: NO)` — the DB password is NOT in `config/database.php`, it lives in `/etc/phlix/env` (the systemd `EnvironmentFiles` for `phlix-server`, found via `systemctl show phlix-server -p EnvironmentFiles`). Source it first: `set -a; . /etc/phlix/env 2>/dev/null; set +a` before invoking `php script.php`.
- **[pattern]** To reproduce an authenticated `/app` state against the live deployment (it redirects to `/app/login` without a token), mint an admin JWT on the hub — query `users WHERE is_admin = 1`, then `ContainerFactory::create($c)->get(JwtHandler::class)->createAccessToken((string)$id)` — capture it, then in Playwright `localStorage.setItem('access_token', <jwt>)` and re-navigate. The UI reads the token from `localStorage` key `access_token`.
- **[pattern]** When diagnosing the virtualized `MediaGrid` scroll/window behavior in Playwright, drive scrolling with `setInterval(() => window.scrollBy(0, …), 8)` (NOT a `requestAnimationFrame` loop) — an rAF-based driver starves the component's own rAF measure and hides the bug. Detect a frozen virtualization window by sampling the inner `.media-grid` `translateY(...)` against `window.scrollY - sizerDocTop` each tick; a growing gap means the window isn't tracking the scroll.
- **[gotcha]** In `MediaGrid.vue` the scroll handler must call `measure()` SYNCHRONOUSLY, not an rAF-deferred `scheduleMeasure` — `requestAnimationFrame` is throttled aggressively during scrolling (notably Firefox), which stalls the deferred measure and freezes the rendered window so the same titles appear to "stay" on screen. Keep `resize` rAF-coalesced; only `scroll` needs the synchronous path. Regression-test it by `vi.stubGlobal('requestAnimationFrame', () => 0)` (returns an id, never fires the callback) then dispatching a `scroll` event and asserting the window still advanced (`translateY` ≠ `translateY(0px)`).
