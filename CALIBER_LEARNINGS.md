# Caliber Learnings

Accumulated patterns and anti-patterns from development sessions.
Auto-managed by [caliber](https://github.com/caliber-ai-org/ai-setup) — do not edit manually.

- **[pattern]** Pure, side-effect-free utilities (sorting/serialization helpers) go in `src/utils/` (or `src/api/`), are exported from `src/index.ts` for consumers, and are unit-tested directly via `it.each` parity tables. When mirroring server behavior (e.g. `SortTitle` article stripping), keep the rule set byte-for-byte in step and assert the shared constant in a test so drift is caught.
- **[pattern]** DOM-free windowing/paging math lives in `src/components/virtual-grid.ts` as pure exported functions tested directly — jsdom has no layout, so `MediaGrid.vue` only measures and delegates. Add new grid logic as a pure helper there with its own test.
- **[fix]** When a `vi.fn` async mock returns DIFFERENT object shapes per branch, `vue-tsc` infers an over-narrow union (members `never[]`) and later branches fail to assign. Fix by widening the mock's return type: `vi.fn(async (endpoint: string): Promise<Record<string, unknown>> => { ... })`. Vitest passes either way; only the typecheck catches this.
- **[gotcha]** In `Player.vue`, the `ended` media event does NOT reliably co-fire a `pause` event (Safari fires `ended` only) — chrome/visibility logic gated on `player.playing` must be pinned explicitly in `onEnded()` (call `clearIdle()` + `showChrome.value = true`), not left to the play/pause watcher. Test by dispatching raw `new Event('play'|'pause'|'ended')` under `vi.useFakeTimers()` and asserting `is-chrome-hidden` on `w.classes()`.
- **[env]** PHP one-shot scripts run over SSH on `hub.phlix` that open a DB connection fail with `SQLSTATE[HY000] [1045] Access denied for user 'phlix'@'localhost' (using password: NO)` — the DB password is NOT in `config/database.php`, it lives in `/etc/phlix/env` (the systemd `EnvironmentFiles` for `phlix-server`, found via `systemctl show phlix-server -p EnvironmentFiles`). Source it first: `set -a; . /etc/phlix/env 2>/dev/null; set +a` before invoking `php script.php`.
- **[pattern]** To reproduce an authenticated `/app` state against the live deployment (it redirects to `/app/login` without a token), mint an admin JWT on the hub — query `users WHERE is_admin = 1`, then `ContainerFactory::create($c)->get(JwtHandler::class)->createAccessToken((string)$id)` — capture it, then in Playwright `localStorage.setItem('access_token', <jwt>)` and re-navigate. The UI reads the token from `localStorage` key `access_token`.
- **[pattern]** When diagnosing the virtualized `MediaGrid` scroll/window behavior in Playwright, drive scrolling with `setInterval(() => window.scrollBy(0, …), 8)` (NOT a `requestAnimationFrame` loop) — an rAF-based driver starves the component's own rAF measure and hides the bug. Detect a frozen virtualization window by sampling the inner `.media-grid` `translateY(...)` against `window.scrollY - sizerDocTop` each tick; a growing gap means the window isn't tracking the scroll.
- **[gotcha]** In `MediaGrid.vue` the scroll handler must call `measure()` SYNCHRONOUSLY, not an rAF-deferred `scheduleMeasure` — `requestAnimationFrame` is throttled aggressively during scrolling (notably Firefox), which stalls the deferred measure and freezes the rendered window so the same titles appear to "stay" on screen. Keep `resize` rAF-coalesced; only `scroll` needs the synchronous path. Regression-test it by `vi.stubGlobal('requestAnimationFrame', () => 0)` (returns an id, never fires the callback) then dispatching a `scroll` event and asserting the window still advanced (`translateY` ≠ `translateY(0px)`).
- **[gotcha]** A PR that edits `src/` but does NOT rebuild the committed `dist/` silently ships STALE compiled code — `vitest`/`vue-tsc` never touch `dist/`, so green checks don't catch it, and consumers (pinned to a git tag, see below) install the old build. v0.44.0 shipped this way and needed a follow-up `chore(dist): rebuild dist for vX.Y.Z` PR. Always `npm run build` and stage `dist/` in the SAME branch/PR as the `src/` change, before committing. Confirm the new code actually landed with `grep -rl <new-symbol> dist/`.
- **[fix]** If a freshly-cut release tag points at a src-only commit and `dist/` was rebuilt in a later follow-up commit, move the tag forward with `git tag -f vX.Y.Z <dist-rebuild-commit> && git push -f origin vX.Y.Z`. This force-move is acceptable ONLY because the tag was not yet consumed by any merged consumer PR — never force-move a tag a consumer already pins.
- **[convention]** `dist/` is committed (tracked, NOT gitignored — verify with `git check-ignore dist`) and consumers `phlix-server/web-ui` + `phlix-hub/web-ui` pin `@phlix/ui` by git tag (`"github:detain/phlix-ui#vX.Y.Z"`), NOT by npm version. A release is incomplete until: bump `package.json` version → update CHANGELOG → `npm run build` + commit `dist/` (same branch) → merge → `git tag vX.Y.Z <squash-merge-commit>` → `git push origin vX.Y.Z` → bump the `#vX.Y.Z` ref in each consumer's package.json. Skipping the tag/consumer-bump means the new UI never ships.
- **[convention]** phlix-ui release tags are LIGHTWEIGHT, not annotated. Create as `git tag vX.Y.Z <commit>` then `git push origin vX.Y.Z` (verify with `git ls-remote --tags origin vX.Y.Z`). Don't "fix" existing tags to `-a` annotated to match other repos.
- **[pattern]** Release/merge flow: branch → `npm run build` + stage dist → commit → `git push -u origin <branch>` → `gh pr create` → `gh pr merge --squash --delete-branch` → `git checkout master && git pull` → tag + push the tag.
- **[env]** Prefix git/gh shell blocks with `unset GITHUB_TOKEN GH_TOKEN` — pushes use the SSH remote (`git@github.com:detain/...`) and a stale token interferes. A caliber pre-commit hook auto-runs on every commit (`caliber: refreshing docs...`) and leaves no extra uncommitted changes — expect it, don't treat its output as an error.
- **[gotcha]** `npm run lint` / `npx eslint` do NOT work in this repo — there is no `eslint.config.js` (flat-config), so every invocation errors out (`ESLint couldn't find an eslint.config.(js|mjs|cjs) file`) regardless of whether eslint is installed. Note `npx eslint ...; echo "exit=$?"` reports exit=0 even on this failure, so don't trust the exit code. The real gates are `npx vue-tsc --noEmit` (typecheck) + `npx vitest run`.
- **[gotcha]** A green `npx vitest run` does NOT mean the build is clean — vitest does not type-check mocks, but `vue-tsc --noEmit` does. Always run the typecheck after editing `.test.ts` files, not just the tests.
- **[gotcha]** The phlix-ui CI workflow (`.github/workflows/ui-ci.yml`) is DORMANT — it only triggers on `workflow_dispatch` (pull_request/push triggers commented out). So `gh pr checks <n>` returns "no checks reported" and `statusCheckRollup` is `[]`. Do NOT poll/wait for CI on phlix-ui PRs — merge once local `vue-tsc --noEmit` + `vitest run` are green.
- **[gotcha]** When converting a helper from raw `fetch` to `ApiClient`, its existing `fetch` mocks BREAK silently (assert mismatch, not a clear error). ApiClient reads `response.status`, `response.headers.get('content-type')`, and `response.json()/.text()` — rebuild the mock as a Response-shaped object: `{ ok, status, headers: { get: () => 'application/json' }, json, text }`.
- **[convention]** `new ApiClient()` with no explicit `tokenStore` defaults to `LocalStorageTokenStore` in the browser, so it auto-sends the logged-in user's Bearer token. Pass an explicit no-op `tokenStore` ONLY when you deliberately want an unauthenticated request.
- **[convention]** The hub's server-claim endpoint `POST /api/v1/server-claims/claim` requires an `Accept-Phlix-Protocol: v1` header IN ADDITION to the Bearer token. The generic `ApiClient` does not send that header, so claim-style hub calls use a dedicated raw-`fetch` helper (e.g. `claimServer`).
- **[convention]** Media-query serialization has ONE source of truth: `buildMediaQuery`/`buildMediaUrl` in `src/api/media-query.ts`. Do NOT add a local query-param builder anywhere — a duplicated serializer silently drifts and drops params. Add new filter params to `buildMediaQuery` only, then have the store use the shared builder for BOTH the request URL and the `cacheKey`.
- **[gotcha]** A FilterBar/store test asserting on `queryParams`/`toQuery` does NOT prove a param reaches the server. The store calls raw `fetch`, not the injected ApiClient — to prove it, `vi.stubGlobal('fetch', fetchMock)`, set the param via a store action, `await s.fetchMedia('')`, then assert `String(fetchMock.mock.calls[0][0])` contains it. For the auth header, seed `localStorage.setItem(ACCESS_TOKEN_KEY, ...)` and read `fetchMock.mock.calls[0][1].headers`.
- **[pattern]** Component tests in `src/pages/**/*.test.ts` inject a `makeClient`/`mountPage` fake (`{ get, post, ... }` of `vi.fn()`) via the `client` prop, plus factory functions for fixture rows. To disambiguate two buttons sharing a label, submit via dispatching a `submit` Event on the `<form>` rather than clicking. To test a `window.open` action, `vi.spyOn(window, 'open').mockReturnValue(null)` and assert the `(url, '_blank', 'noopener,noreferrer')` call; assert a button's disabled state via `btn.props('disabled')`.
